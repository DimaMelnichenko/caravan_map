# Протокол обмена данными Caravan Map

## Обзор

Проект использует гибридную архитектуру:
- **REST API** — для CRUD операций (создание, чтение, обновление, удаление)
- **WebSocket** — для синхронизации состояния в реальном времени

## REST API Endpoints

### Мир

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/load` | Загрузка всех данных мира |
| POST | `/api/viewport` | Получить данные для viewport |

### Города

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/cities` | Создать/обновить город |
| DELETE | `/api/cities/:id` | Удалить город |

### Маршруты

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/routes` | Создать/обновить маршрут |
| DELETE | `/api/routes/:id` | Удалить маршрут |
| POST | `/api/routes/:id/init-caravans` | Инициализировать караваны |

### Караваны

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/caravans` | Получить все караваны |
| POST | `/api/caravans/:id/update` | Обновить позицию каравана |

### Страны

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/countries` | Создать/обновить страну |
| DELETE | `/api/countries/:id` | Удалить страну |

### Границы

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/borders` | Создать/обновить границу |
| DELETE | `/api/borders/:id` | Удалить границу |

### Экономика

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/city-economy` | Сохранить экономику города |
| POST | `/api/save-inventory` | Сохранить инвентарь складов |

## WebSocket Protocol

### Сообщения Клиент → Сервер

#### viewport_update
Обновление видимой области клиента
```json
{
  "type": "viewport_update",
  "viewport": {
    "x": 100,
    "y": 200,
    "width": 800,
    "height": 600
  }
}
```

#### action
Действие игрока (создание города и т.д.)
```json
{
  "type": "action",
  "action": {
    "type": "create_city",
    "name": "Москва",
    "x": 500,
    "y": 300
  }
}
```

#### ping
Проверка соединения
```json
{
  "type": "ping",
  "timestamp": 1234567890
}
```

### Сообщения Сервер → Клиент

#### full_state
Полное состояние мира при подключении
```json
{
  "type": "full_state",
  "data": {
    "tick": 0,
    "cities": [...],
    "routes": [...],
    "countries": [...],
    "borders": [...],
    "caravans": [...],
    "items": [...]
  }
}
```

#### tick
Сигнал тика игрового цикла
```json
{
  "type": "tick",
  "data": {
    "tick": 123
  }
}
```

#### caravans_delta
Дельта изменений караванов
```json
{
  "type": "caravans_delta",
  "data": [
    {
      "id": 1,
      "x": 150.5,
      "y": 200.3,
      "progress": 45.2,
      "state": "moving"
    }
  ]
}
```

#### state_delta
Дельта состояния (картаваны в viewport)
```json
{
  "type": "state_delta",
  "data": {
    "tick": 123,
    "caravans": [...]
  }
}
```

#### caravan_arrived
Караван прибыл в город
```json
{
  "type": "caravan_arrived",
  "data": {
    "id": 1,
    "route_id": 5,
    "to_id": 10
  }
}
```

#### trade_complete
Торговля завершена
```json
{
  "type": "trade_complete",
  "data": {
    "id": 1,
    "from_id": 10,
    "to_id": 15,
    "progress": 0
  }
}
```

#### pong
Ответ на ping
```json
{
  "type": "pong",
  "timestamp": 1234567890
}
```

## Синхронизация

### Частота обновлений
- **Игровой тик**: 1 секунда
- **Отправка состояния**: 100 мс (10 раз в секунду)
- **Обновление viewport**: 500 мс

### Интерполяция
Клиент использует интерполяцию с задержкой 100 мс для плавного движения караванов.

### Переподключение
При разрыве соединения клиент пытается переподключиться с экспоненциальной задержкой:
- 1 попытка: 1 сек
- 2 попытка: 2 сек
- 3 попытка: 4 сек
- ...
- Максимум: 30 сек

## Безопасность

- Валидация всех входящих данных на сервере
- Ограничение размера запросов (10 MB)
- CORS для контроля доступа

## Производительность

- Interest management: отправка только видимых сущностей
- Дельта-компрессия: отправка только изменений
- Пакетная отправка обновлений
